## 2. Bug — descrição de núcleo não atualizava sem F5 (FEITO)

**Sintoma:** editar Descrição do núcleo na Constelação → Salvar → reabrir editor mostrava valor antigo até F5.
A História da entidade tinha o mesmo sintoma.

**Causas e correções** (`public/js/constelacao.js`):
- **História/Reputação (entidade):** GETs `/nodes/:id/historia` e `/reputacao` **sem anti-cache** → o
  navegador servia resposta cacheada. **Fix:** `?_=${Date.now()}` (padrão já usado no snapshot `:67/:310`).
- **Núcleo:** o editor lê `o.descricao` do array `orbes`; o `recarregar()` não refletia a tempo. **Fix:**
  **update otimista** no save do `cf-salvar` — atualiza `o.nome/descricao/cor/escala/tarot` localmente logo
  após o PUT, antes do `fechar()`. Confirmado pelo usuário: **funcionou**.

## 3. profile.html — "Sistema" sempre "Mago M20" (FEITO)

**Causa:** `"Sistema: Mago M20"` hardcoded no `profile.js`; o endpoint `/auth/dashboard-resumo` nem buscava
o sistema. **Pegadinha:** `cronicas` tem `sistema_id` (uuid, FK→`sistemas`) **E** `sistema` (varchar legado,
slug ex. `mago_m20`). As crônicas do usuário têm o valor no **legado** com `sistema_id` nulo.

**Fix** (`controllers/authController.js` `dashboardResumo`): dois `LEFT JOIN` (não `OR`, p/ não duplicar
linha) + `COALESCE(s_id.nome, s_slug.nome, c.sistema) AS sistema_nome`. Front consome com `escapeHTML`.
Confirmado: **funcionou**.

## 5. Tabuleiro — botões de gestão de imagem (FEITO)

**Pedido:** Ajustar/Remover fundo só devem aparecer quando há imagem. **UX escolhida pelo usuário:**
**desabilitar (cinza)**, não esconder (barra estável).

**Causa do "não escondia":** `hidden` é sobrescrito por `.btn { display: inline-flex }` (regra de autor vence
o UA stylesheet). **Fixes:**
- `global_ui.css`: novo `.btn:disabled` (opacidade/cursor/`pointer-events:none`) + reset pontual
  `.board-fundo-ctrl[hidden] { display: none !important; }` (conserta os controles de zoom/opacidade).
- `controle_mundo.html`: Ajustar/Remover (`.board-fundo-img-ctrl`) passam de `hidden` para `disabled`.
- `controle_mundo.js`: `sincronizarControlesFundo()` faz `el.disabled = !temImagem`; corrigida a ORDEM —
  era chamado antes de `boardState` carregar (refletia o tabuleiro anterior); agora chamado **depois** nos
  dois ramos de `abrirBoard` + após upload.
